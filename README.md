# 工程施工自主檢查表智能生成系統

這是一個基於 Python 開發的完整 MVP（最小可行產品）系統，旨在幫助工程現場人員自動化處理白板照片，並智能生成規範的 Word 格式施工檢查表。

本系統結合了 **PaddleOCR** 的強大中文手寫辨識能力，以及 **Google Gemini LLM** 的智能糾錯與術語標準化能力，將繁瑣的文書作業自動化，大幅提升工程管理的效率與準確度。

## 🌟 核心功能

* **一鍵上傳與處理**：支援直接上傳包含工地照片的 ZIP 檔案，系統自動解壓縮並讀取資料夾結構。
* **智能圖像處理**：自動裁切、壓縮照片，並調整為適合嵌入 Word 表格的統一長寬比。
* **高精度文字辨識**：內建 PaddleOCR 引擎，針對繁體中文與工地手寫白板字跡進行辨識。
* **LLM 智能糾錯 (代理檢閱者)**：利用 Gemini API 進行「代理檢閱」，將 OCR 辨識出的亂碼或錯字（如把「鋼筋」辨識成「銅筋」），透過工程術語提示詞（Prompt）自動糾正，並標準化為結構化資料。
* **Word 文檔自動生成**：讀取包含佔位符（如 `{{image_1}}`、`{{date}}`）的 Word 範本，自動填入照片與辨識結果。
* **完整歷程記錄**：內建 SQLite 資料庫，完整記錄每一次的處理歷程、辨識置信度與耗時。

## 🏗️ 系統架構與技術選型

本系統採用前後端分離的現代化架構，所有元件均使用 Python 生態系中最適合的工具：

| 元件 | 技術選型 | 說明 |
| --- | --- | --- |
| **前端介面** | Streamlit | 提供使用者友善的 Web 介面，支援拖曳上傳與實時進度顯示，非常適合快速原型開發。 |
| **後端 API** | FastAPI | 高效能非同步 API 框架，負責協調檔案解壓縮、OCR 運算與 LLM 調用。 |
| **資料庫** | SQLite | 輕量級關聯式資料庫，無需額外配置即可記錄專案、範本路徑與產製歷程。 |
| **圖像處理** | Pillow (PIL) | 輕巧高效的圖像處理套件，負責照片的壓縮與尺寸調整。 |
| **OCR 模組** | PaddleOCR | 開源且免費，對繁體中文及手寫字的辨識能力極佳。 |
| **LLM 糾錯** | Google Gemini API | 提供強大的語意理解與糾錯能力，大幅提升最終輸出的準確率。 |
| **文檔生成** | python-docx | 原生支援 Word 文檔操作，可精確替換佔位符並嵌入圖像。 |

## 🚀 快速開始

### 前置需求

* Python 3.11+
* Docker & Docker Compose (推薦)
* Google Gemini API Key (用於 LLM 智能糾錯)

### 方法一：使用 Docker 部署 (推薦)

使用 Docker 部署可以確保環境的一致性，並避免繁瑣的依賴安裝。

1. **配置環境變數**：
   複製 `.env.example` 並重新命名為 `.env`，填入您的 API Key：
   ```bash
   cp .env.example .env
   # 編輯 .env 檔案，填入 GEMINI_API_KEY
   ```

2. **啟動服務**：
   ```bash
   docker-compose up -d --build
   ```

3. **存取系統**：
   * 前端介面 (Streamlit): http://localhost:8501
   * 後端 API (FastAPI): http://localhost:8000
   * API 文件 (Swagger UI): http://localhost:8000/docs

### 方法二：本地開發環境安裝

如果您希望在本地進行開發或除錯，請依照以下步驟安裝：

1. **建立虛擬環境**：
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows 請使用 venv\\Scripts\\activate
   ```

2. **安裝依賴套件**：
   ```bash
   pip install -r requirements.txt
   ```

3. **啟動後端 API**：
   ```bash
   # 在一個終端機視窗中執行
   uvicorn backend.api:app --reload --host 0.0.0.0 --port 8000
   ```

4. **啟動前端介面**：
   ```bash
   # 在另一個終端機視窗中執行
   streamlit run app/main.py
   ```

## 📖 使用指南

### 1. 準備上傳資料

系統接受 ZIP 格式的壓縮檔。請將工地白板照片放置於資料夾中，並將資料夾壓縮為 ZIP 檔。
建議的資料夾命名方式為「專案名稱_樁號_位置」（例如：`600樁號_3T4上`）。

### 2. 系統操作流程

1. 開啟瀏覽器進入系統首頁 (http://localhost:8501)。
2. 在左側邊欄確認或輸入您的 Google Gemini API Key。
3. 於「上傳處理」頁籤，將準備好的 ZIP 檔案拖曳至上傳區。
4. 勾選「啟用 LLM 智能糾錯」（強烈建議啟用以提升準確率）。
5. 點擊「🚀 開始處理」按鈕。
6. 系統將自動執行解壓縮、圖像處理、OCR 辨識、LLM 糾錯與 Word 文檔生成。
7. 處理完成後，您可以直接下載生成的 Word 檢查表或包含所有結果的 ZIP 壓縮檔。

### 3. 處理歷史查詢

切換至「處理歷史」頁籤，您可以查看過去所有的處理記錄，包含專案名稱、處理狀態、上傳時間與總耗時。這有助於追蹤專案進度與系統效能。

## 🛠️ 自訂 Word 範本

系統預設提供了一個標準的 Word 檢查表範本 (`templates/checklist_template.docx`)。您可以根據實際需求修改此範本，只需確保保留以下佔位符：

* `{{date}}`：檢查日期
* `{{location}}`：檢查地點
* `{{pile_number}}`：樁號/位置
* `{{image_1}}`, `{{image_2}}`, `{{image_3}}`：照片插入位置
* `{{item_1}}`, `{{spec_1}}`, `{{result_1}}`：檢查項目、規格與結果（支援多項，依序遞增數字）

## ⚠️ 注意事項與未來展望

* **OCR 效能**：首次執行 PaddleOCR 時，系統會自動下載模型檔案，可能需要一些時間。
* **API 限制**：使用 Gemini API 時請注意配額限制。若需處理大量資料，建議申請付費方案或替換為其他商業 API (如 OpenAI 或 Azure)。
* **排版優化**：在自訂 Word 範本時，應注意表格排版，避免因內容過少而導致頁面空白，尤其是連續的項次應盡量保持在同一頁面，以確保內容的連貫性和版面的整潔。
* **進階部署**：目前系統使用 SQLite 作為本地資料庫，適合初期測試與小規模部署。若要正式上線處理高併發請求，建議將資料庫遷移至 PostgreSQL，並將前端改為 React 或 Vue.js 搭配 TailwindCSS 製作響應式網頁。
